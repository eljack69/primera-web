Features

Built for the way you actually debug.

Drop in our SDK once and get traces, logs, metrics, and alerts — all queryable in one place, in milliseconds.

Sub-second queries

Columnar storage and adaptive indexing return results in under 300ms across billions of rows.

Zero-config tracing

Auto-instrument Node, Go, Python, and Rust. No vendor lock-in, no manual spans, no YAML.

AI-powered alerts

Anomaly detection that learns your baselines. Get paged only when it actually matters — not every spike.

Live tail with regex

Stream production logs in real time. Filter with regex, jump to traces, no terminal gymnastics required.

OpenTelemetry native

Speaks OTLP, Prometheus, and StatsD out of the box. Bring your existing pipeline, switch in minutes.

Pay only for what you use

Volume-based pricing, no per-host or per-seat fees. Spend caps and forecasts keep bills predictable.

How it works

From install to insight in three steps.

01 — Install

Add the SDK

$ npm install @pulse/sdk

One package. Works with any framework — Next.js, Express, Fastify, Hono, NestJS.

02 — Initialize

Call init once

// app entry
import { pulse } from '@pulse/sdk';

pulse.init({ key: 'pk_live_xxx' });

Auto-instruments HTTP, DB, cache, and queue calls. No manual spans to wire up.

03 — Observe

See metrics flow in

# first traces in seconds
GET /api/checkout      142ms  200
GET /api/products      28ms   200
POST /api/webhook      91ms   200

Open the dashboard. Latency, errors, and traces stream live with sub-second freshness.
